Fent proves

Una manera molt ràpida de crear un servidor de correu és gràcies a poste.io. Aquesta web ens ofereix una demo per a fer proves i demés. Gràcies a aquesta demo som capaços de crear mails i fer diverses proves, encara que sigui de manera temporal :

Veiem com és possible rebre missatges desde aquests comptes a el compte de l’institut, per exemple

Però a l’hora de respondre, no deixa, ja que el domini creat no existeix :

Provem a INTE

  1. Instal·lem Docker a inte
  2. Creem un volum, amb la comanda docker volume create. Exemple : docker volume create volum_mail_server
  3. Engeguem el servei amb la comanda docker run. Exemple : docker run -p 443:443 -e TZ=Europe/Andorra -v volum_mail_server:/data –name “MailSanJorEva” -h “sanjoreva.io” -t analogic/poste.io

Com es veu, fem ús del servidor de poste.io

Un cop tenim el contenidor creat, simplement tenim que iniciar-lo amb la comanda docker start + el nom del contenidor

Si posem la IP del servidor de correu, que és inte, entrarem a webmail, és a dir, el client, des d’on podem accedir amb algun usuari del nostre domini

Imatge on es veu que es poden enviar missatges, però que no es pot contestar (com a l’exemple de la demo)

Creació d’un nou domini amb curl

Fent ús de la demo de poste.io, podem utilitzar l’usuari admin que ens proporciona per a crear un nou domini gràcies a RestAPI amb la següent comanda :

curl -v -k -u admin@poste.io:admin -d ‘name=jordialonso.org’ https://demo.poste.io/admin/api/v1/domains
